## Description

Pecha web app tool for collecting and managing Speech-to-Text (STT) data for Tibetan language processing. Ideally used for managing large-scale transcription projects with multiple groups and departments.

## Features

- **Task Management**: Assign, submit, and review transcription tasks.
- **Workflow Roles**: Distinct workflows for Transcribers, Reviewers, and Final Reviewers.
- **Automated Reporting**:
  - Detailed performance metrics (audio duration, syllable count, CER).
  - Department and Group-level aggregations.
- **Payment Calculation**: Automated calculation of payments (Rs.) based on reviewed by 25th of each month, It includes submitted audio duration, syllable count, CER.

## Architecture Highlights

### Timezone Strategy
**The application operates strictly in Indian Standard Time (IST).**
- All timestamps in the database are stored as UTC but represent IST values (e.g., 12:00 IST is stored as 12:00 UTC).
- The application explicitly shifts dates using `istCurrentTime()` logic to align reporting days with the Indian workday.

### Reporting Engine
- Reports are generated by fetching raw data and processing it in-memory (`reportEngine.ts`).
- Supports complex filtering by date range (IST) and group.
- **Metrics**: Calculates Charater Error Rate (CER) using Levenshtein distance for quality assurance.

## Install

Requires `node` (>=18) and `npm` installed.

1. Install dependencies:
   ```bash
   npm install
   ```

2. Run the development server:
   ```bash
   npm run dev
   ```

## Environment Variables

Create a `.env` file in the root directory:

- `DATABASE_URL`: PostgreSQL connection string.
